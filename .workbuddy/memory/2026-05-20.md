## index.html 双轨旅程视图改造（v1.2）

### 背景
原 index.html 仅平铺 8 个 AI 场景卡片，销售体系无法一眼看出每个场景在销售转化路径中承担的角色。豆爸要求引入"销售链路 / 用户旅程"视角，且销售培训和用户转化是两条不同路径，需要双轨呈现。豆爸明确否定 SVG 方案（矢量精度不够），采用 HTML + CSS 卡片矩阵实现。

### 信息架构（双轨模型）
- **外轨（客户转化旅程）**：线索 → 接触 → 跟进 → 提案 → 成交 → 复购（6 节点）
- **内轨（销售成长旅程）**：入职准备 → 模拟实训 → 实战赋能 → 复盘沉淀（4 阶段）
- **资产飞轮**：外轨成交 → 内轨沉淀 → 外轨复用（4 步骤闭环）
- 8 场景在两轨中复用映射：01 知识库 / 02 话术参谋 / 04 复盘沉淀 / 06 模拟培训 / 08 方案生成 同时出现在两轨

### 实施
- `index.html`（113→435 行）：
  - `<header>` 后插入浮动 TOC `<nav class="toc-floating">`（5 锚点）
  - `<main>` 内 5 板块：旅程总览（双 track-row + 飞轮提示）/ 客户转化全链路（外轨 6 卡片）/ 销售成长全链路（内轨 4 卡片）/ 资产飞轮（4 步骤 + 资产清单 + 复利清单）/ 全部场景索引（原 8 卡片下移）
  - `<script>` 扩展：
    - scenes 数组每条新增 `stages[]`（外轨节点）和 `tracks[]`（内轨节点）字段
    - 新增 `clientStages`（6 节点 × 4 字段：customer/pain/ai/metric）
    - 新增 `salesStages`（4 阶段 × 4 字段：sales/pain/ai/metric）
    - 新增 `renderPills()` / `renderStageCards()` 函数
    - 浮动 TOC 接 IntersectionObserver 滚动高亮（rootMargin: '-40% 0px -50% 0px'）
- `shared/css/styles.css`（797→1157 行，追加约 360 行）：
  - 新增 6 个轨道色 Token：`--track-client`（蓝）/ `--track-sales`（琥珀）+ 浅底/深字派生色
  - `.journey-section` / `.track-row.client/.sales` / `.stage-pill`（hover 上浮 + ::before/::after 伪元素绘制水平箭头）
  - `.stage-grid.client`（3 列）/ `.stage-grid.sales`（2 列）+ `.stage-card`（顶部 3px 轨道色边）
  - `.stage-fields`（88px label + 1fr 文本）+ `.stage-field-text.metric/.pain` 状态色
  - `.stage-scenes` + `.stage-scene-chip`（反向链接到对应 scene demo）
  - `.flywheel` 双列布局 + `.flywheel-step.t1/t2/t3/t4`（4 步骤不同主色 + ::before "↓"连接符）
  - `.toc-floating` 固定定位 + backdrop-filter 毛玻璃 + `.toc-item.active` 高亮
  - 三级响应式断点：1100（双栏收一栏 + 隐藏 TOC）/ 900（堆叠）/ 640（单列）

### 关键决策
- **媒介选择**：放弃 SVG 改用 HTML + CSS 卡片矩阵；节点连接箭头用 CSS 伪元素绘制
- **复用率**：外轨 6 节点 × 内轨 4 阶段共 10 个挂载点，8 场景被分配 14 次（5 场景双轨复用），证明双轨视图非生硬切割
- **TOC 仅大屏显示**：1100px 以下隐藏，避免移动端干扰
