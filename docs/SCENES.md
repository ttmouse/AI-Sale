# AI-Sale 八大场景细化方案

> 适用行业：智能炒菜机 / 商用厨具 / 非标餐饮设备
> 核心命题：AI 不替代销售，专注全链路提效、标准化、可复制

## 场景 1：销售随身智能知识库 & 产品问答助手

### 业务痛点
- 产品型号多（YC-1000 / YC-2000 / YC-3000 / 定制款）、配置杂、适配场景多
- 卖点、参数、报价逻辑、常见问题，新人记不住，老销售也要翻资料
- 客户一问"这台机器能做多少人的菜？""食堂用哪一款？""和XX品牌比优势在哪？"就卡壳
- 培训成本高：新人背资料要 2-3 周才能独立应对客户

### AI 落地能力
- **产品参数问答**：型号、功率、容量、价格、适用人数、占地面积
- **场景适配推荐**：输入门店类型+面积+单量，自动推荐适配机型+数量+配套方案
- **竞品对比**：客户提到竞品时，自动输出差异化优势
- **报价逻辑**：根据配置自动计算报价区间
- **常见问题应答**：售后政策、质保期限、安装要求等

### Demo 展示形态
**界面**：类 ChatGPT 对话界面，底部加"快速场景入口"按钮（食堂 / 连锁 / 创业小店 / 酒店厨房 / 竞品对比）

**示例交互**：
```
销售输入：300 平川菜馆，日均 300 单，预算 8 万内

AI 输出：
━━━━━━━━━━━━━━━━━━━━━━
推荐配置：
• YC-3000 智能炒菜机 × 2 台  
  （单台产能：200 单/日，双台冗余覆盖）
• 配套：排烟系统 + 专用操作台 + 电源改造

预估报价：6.8 - 7.5 万
回本周期：约 8-10 个月（按日均毛利 300 元计）

同类案例：
• 成都「川味轩」（280 平，2 台 YC-3000，3 个月回本）
• 杭州「辣妹子」（320 平，上线后后厨人力减少 40%）

下一步建议：安排试菜体验，客户转化率提升 60%
━━━━━━━━━━━━━━━━━━━━━━
```

### 数据准备
| 数据类型 | 内容 | 数量 |
|---------|------|------|
| 产品知识库 | 型号、参数、价格、卖点 | 4-6 个主型号 |
| 场景适配规则 | 餐饮类型→机型→数量→配套 | 8-10 种场景 |
| 竞品对比表 | 主要竞品型号、价格、优劣势 | 3-5 个竞品 |
| 案例库 | 真实/模拟客户案例 | 10-15 个 |
| FAQ 库 | 常见客户问题及标准答案 | 50-100 条 |

### 技术栈
- **前端**：纯 HTML + Tailwind（对话界面）
- **AI 层**：RAG（向量检索 + LLM 生成），或关键词匹配 + LLM 回答
- **数据存储**：JSON 静态文件即可（Demo 阶段）

### 核心亮点
- 销售不用背参数，输入场景自动出方案
- 带案例、带回本周期、带下一步建议，不只是问答
- 新人 1 天上手，替代 2-3 周资料背诵期

### 预期价值
- 新人培训周期从 2-3 周 → 2-3 天
- 客户专业问题响应速度从"我查一下"→秒级
- 销售方案输出标准化，减少个人水平差异

### 落地难点
- 产品知识库需要持续更新（新品上市、价格调整）
- 案例库需要销售团队配合提供真实数据
- **建议**：先以静态知识库起步，逐步接入动态数据

## 场景 2：企微聊天实时随行参谋（半自动话术推荐）

### 业务痛点
- 销售临场不会接话、不会挖需求、不会引导成交阶段
- 只会干发资料（产品册、报价单），没有引导力
- 直接 AI 自动回复怕说错商务政策、怕语气不对
- 高客单销售需要"人"的温度，不能完全自动化

### AI 落地能力
- **不自动发消息**，只做侧边栏/弹窗建议
- 实时识别聊天上下文 + 客户所处阶段（P1 认知 → P2 兴趣 → P3 评估 → P4 决策）
- 按阶段推荐话术：
  - P1：需求挖掘话术（怎么问出真实需求）
  - P2：方案引导话术（怎么引导试菜/上门）
  - P3：异议处理话术（怎么应对砍价、售后担忧）
  - P4：促成话术（怎么催单、怎么制造紧迫感）
- 适配手机 + 电脑企微使用习惯

### Demo 展示形态
**界面**：双栏布局
- **左 60%**：模拟企微聊天窗口（销售 vs 客户对话）
- **右 40%**：AI 参谋侧边栏（实时显示当前对话状态 + 推荐话术卡片）

**示例交互**：
```
【客户】：你这个机器多少钱一台？

【AI 参谋侧边栏】：
━━━━━━━━━━━━━━━━━━━━━━
客户阶段判断：P2 兴趣期 → P3 评估期
当前意图：询价（可能已进入比价阶段）

推荐话术（3 选 1）：

① 转移焦点法 ⭐ 推荐度 95%
"价格根据配置不同，从 3 万到 8 万都有。
  我先了解下您的场景，给您配个最合适的方案？"
→ 把价格话题转回需求挖掘

② 锚定报价法  推荐度 80%
"我们标准款 4.98 万，本月活动价 4.28 万。
  您主要是做快餐还是炒菜？"
→ 给价格锚点 + 继续挖需求

③ 案例佐证法  推荐度 70%
"附近 XX 川菜馆刚定了 2 台，总投资 7 万，
  3 个月就回本了。我先给您安排试菜？"
→ 用案例降低价格敏感度

【销售点击① → 自动填入输入框 → 可编辑后发送】
━━━━━━━━━━━━━━━━━━━━━━
```

### 数据准备
| 数据类型 | 内容 | 数量 |
|---------|------|------|
| 话术库 | 按场景分类的话术模板 | 50-100 条 |
| 客户阶段定义 | P1-P4 判定规则 | 4 个阶段 |
| 模拟对话剧本 | 完整对话流程 | 10-15 组 |
| 异议分类 | 价格/售后/竞品/时机等 | 6-8 类 |

### 技术栈
- **前端**：React/Vue（双栏布局 + 实时状态同步）
- **AI 层**：LLM 分析对话上下文 + 意图识别 + 话术推荐
- **状态管理**：对话阶段机（P1→P2→P3→P4）

### 核心亮点
- **半自动辅助**：AI 建议、销售决策、人来做最终判断
- 实时识别客户阶段，推荐最匹配的话术
- 3 套话术可选，销售根据客户关系灵活选择

### 预期价值
- 新人销售话术水平从"只会发资料"→"会引导、会挖需求"
- 客户转化率预计提升 20-30%
- 销售不用背话术，AI 实时推送

### 落地难点
- 话术库需要根据实际业务持续优化
- 需要抓取企微聊天记录（需企业微信会话存档权限）
- **建议**：Demo 阶段用模拟对话，落地时对接企微 API

## 场景 3：客户分层标签 + 生命周期 SOP 智能跟进

### 业务痛点
- 销售跟进随性、看心情、看客户"颜值"挑着跟
- 长周期客户（1-2 个月）容易遗忘、断联、流失
- 没有统一的跟进节奏：什么时候该回访、什么时候该发案例、什么时候该促单
- 沉睡线索大量浪费：只追大客户，中小线索直接放弃

### AI 落地能力
- **自动标签分层**：
  - 行业标签：快餐 / 火锅 / 食堂 / 酒店 / 私房菜
  - 规模标签：小店（<100 平）/ 中型（100-300 平）/ 大型（>300 平）
  - 意向标签：P1 认知 / P2 兴趣 / P3 评估 / P4 决策 / P5 成交 / P0 沉睡
  - 预算标签：<3 万 / 3-5 万 / 5-10 万 / >10 万
- **SOP 自动编排**：
  - 新客户：Day 1 欢迎 + 需求了解 → Day 3 方案推送 → Day 7 试菜邀约 → Day 14 促单 → Day 30 沉睡唤醒
  - 不同阶段推送不同内容（案例、报价、限时活动、客户见证）
- **智能提醒**：到时间点自动提醒销售跟进，预制文案一键发送
- **沉睡唤醒**：自动识别 30 天未跟进客户，生成唤醒话术

### Demo 展示形态
**界面**：CRM 看板式界面
- **顶部**：客户分层筛选（S/A/B/C/沉睡）+ 行业/规模筛选项
- **主体**：客户卡片网格（带客户画像、当前阶段、下次跟进时间、AI 推荐动作）
- **右侧抽屉**：单个客户的 SOP 时间线（已执行 / 待执行 / 已跳过）

**示例交互**：
```
【客户卡片：张老板 - 川味轩】
━━━━━━━━━━━━━━━━━━━━━━
标签：川菜馆 | 280 平 | P3 评估期 | 预算 5-8 万
最近动态：3 天前试菜完成，反馈"味道不错，考虑中"

AI 建议下一步：
⏰ 今天 14:00 跟进（试菜后第 3 天，黄金跟进窗口）
推荐动作：发送"限时活动"+"同类客户案例"
预制文案：[一键发送]
"张老板，试菜后还满意吗？
  这月我们有个老客户推荐活动，
  成都 XX 川菜馆也是试菜后 3 天定的，
  现在后厨省了 2 个人力。
  您看这周能定下来吗？"
━━━━━━━━━━━━━━━━━━━━━━
```

### 数据准备
| 数据类型 | 内容 | 数量 |
|---------|------|------|
| 客户数据 | 模拟客户档案 | 30-50 个 |
| 标签体系 | 行业/规模/意向/预算 | 4 维度 |
| SOP 模板 | 不同阶段的跟进动作 | 5-8 个阶段 |
| 预制文案 | 各阶段的标准跟进话术 | 20-30 条 |
| 跟进历史 | 模拟的已执行动作 | 100-200 条 |

### 技术栈
- **前端**：React + 状态管理（Redux/Zustand）
- **后端**：Node.js/Python + 数据库（SQLite/PostgreSQL）
- **AI 层**：LLM 生成个性化跟进文案
- **定时任务**：模拟 SOP 触发（Demo 用按钮触发，落地用 Cron）

### 核心亮点
- 客户分层可视化，一眼看出谁在哪个阶段
- AI 自动编排跟进节奏，销售不用记、不用想
- 沉睡客户一键唤醒，盘活存量线索

### 预期价值
- 客户跟进率从"挑着跟"→100% 覆盖
- 沉睡线索盘活率预计提升 30-50%
- 销售跟进效率提升 2-3 倍

### 落地难点
- 需要对接 CRM/SCRM 系统获取客户数据
- SOP 需要业务部门参与定义
- **建议**：先定义 1-2 个核心 SOP 流程跑通，再扩展

## 场景 4：销售聊天记录 AI 复盘 & 优秀打法沉淀

### 业务痛点
- 老销售好的话术、跟进逻辑藏在个人脑子里，离职就带走
- 差的销售一直犯错没人纠正，同样的问题反复犯
- 公司没有统一销售话术库，每次培训都是重新来
- 成交/流失原因不明：为什么这个客户成了？为什么那个客户跑了？

### AI 落地能力
- **抓取聊天记录**：对接企微会话存档，导出销售与客户的完整对话
- **成交案例拆解**：
  - 识别成交客户的沟通路径（从第一话到成交的关键节点）
  - 提炼"金牌话术"：哪些话打动了客户
  - 标注关键动作：什么时候约的试菜、什么时候给的报价
- **流失案例复盘**：
  - 标注"流失点"：哪句话说错了、哪个动作做错了
  - 分析流失根因：价格？时机？竞品？需求不匹配？
- **自动沉淀**：
  - 生成"标准话术卡片"（场景 + 客户问题 + 标准回答）
  - 生成"销售 SOP 培训教材"（按阶段整理最佳实践）

### Demo 展示形态
**界面**：三步流程——上传 → 分析 → 报告
- **上传区**：拖拽聊天记录文件（支持 txt / 截图模拟）
- **分析页**：进度动画 + 关键节点标注
- **报告页**：左右分栏
  - 左侧：原始聊天记录（AI 高亮标注关键语句）
  - 右侧：AI 分析批注（成功因素 / 流失点 / 改进建议）

**示例交互**：
```
【分析报告：客户"李老板"— 火锅城】
━━━━━━━━━━━━━━━━━━━━━━
📊 对话概览
总轮次：23 轮 | 周期：18 天 | 结果：成交（2 台 YC-3000，总价 8.6 万）

⭐ 成功因素（3 个）
1. 第 3 轮：销售主动约试菜（"您下周三方便吗？我带上机器过去"）
   → 试菜是成交的关键转折点，成交客户 78% 都经历了试菜
2. 第 12 轮：客户说"贵"，销售用"人力成本"换算（"一台机器省 2 个厨师，
   一年省 12 万"）→ 成功转移价格焦点
3. 第 18 轮：制造紧迫感（"这月活动价截止 30 号"）→ 促单成功

⚠️ 风险点（1 个）
第 7 轮：销售说"我们的产品是最好的"
→ 过于绝对，如果客户之前用过竞品，容易引起反感
建议改为："根据您的场景，我们的方案适配度比较高"

📋 沉淀产出
• 话术卡片："人力成本换算法"（已入库）
• 关键动作：试菜邀约最佳时机（第 3 轮，兴趣期）
• SOP 更新：评估期 → 决策期 的促成策略
━━━━━━━━━━━━━━━━━━━━━━
```

### 数据准备
| 数据类型 | 内容 | 数量 |
|---------|------|------|
| 成交案例对话 | 完整聊天记录 | 3-5 组 |
| 流失案例对话 | 完整聊天记录 | 3-5 组 |
| 复盘模板 | 分析框架 | 1 套 |
| 话术卡片模板 | 沉淀格式 | 1 套 |

### 技术栈
- **前端**：文件上传 + 高亮展示 + 分栏对比
- **AI 层**：LLM 长文本分析（支持 10k-100k token）
- **存储**：话术卡片库（可搜索、可分类）

### 核心亮点
- 成功案例自动提取"可复制的打法"
- 流失案例标注"关键失误点"，避免重复犯错
- 自动沉淀为公司资产，不因人员流动而丢失

### 预期价值
- 销售培训从"靠经验传承"→"靠数据沉淀"
- 新人成长速度提升 50%
- 公司销售能力从"个体化"→"标准化"

### 落地难点
- 需要企微会话存档权限
- 聊天记录涉及隐私，需要合规处理
- **建议**：Demo 用模拟数据，落地时脱敏处理

## 场景 5：客户个性化信息智能应答（对接内部业务库）

### 业务痛点
- 普通 AI 只能答通用知识（产品参数、报价逻辑）
- 客户问个性化问题："我的货什么时候到？""售后工程师是谁？"
  "试菜预约在几号？""我定的定制款进度怎么样了？"
- 销售要去 ERP/OMS/CRM 里查，效率低、容易查错
- 客户体验差：同一个问题问三遍，每次答案不一样

### AI 落地能力
- **对接内部系统**：ERP（订单）、OMS（物流）、CRM（客户档案）、售后系统
- **自然语言查询**：客户/销售用大白话提问，AI 自动识别意图 → 调用对应 API → 返回准确答案
- **多轮对话支持**：
  - 客户："我的订单到哪了？"
  - AI："您的订单 #20240518001 已于 5 月 19 日发货，预计 5 月 22 日到达杭州"
  - 客户："能改成送到我另一个地址吗？"
  - AI："可以帮您申请改地址，新地址请提供：..."
- **权限控制**：不同角色看到不同的数据范围（销售看自己的客户，主管看团队客户）

### Demo 展示形态

## 场景 6：AI 新人模拟对练 & 销售岗前培训

### 业务痛点
- 线下培训成本高：请老销售讲课、安排场地、占用工作时间
- 新人没人带，不敢跟真实客户聊，成长慢
- 老销售带新人积极性低（影响自己业绩）
- 培训效果难评估：听完课不代表会实战

### AI 落地能力
- **AI 扮演客户**：
  - 挑剔型："你们的机器比 XX 品牌贵 2 万，凭什么？"
  - 犹豫型："我再考虑考虑，过两个月再说"
  - 急单型："我下周就要开业，能马上发货吗？"
  - 技术型："电机功率多大？能不能连续工作 8 小时？"
- **实战对练**：新人跟 AI 客户进行 10-20 轮对话模拟
- **AI 点评**：
  - 三维评分：话术专业度 / 引导力 / 成交概率
  - 逐句点评：哪句话说得好、哪句可以改进
  - 标准答案：给出最优回应示范
- **批量培训**：同时培训 10 个新人，不需要老销售在场

### Demo 展示形态
**界面**：角色扮演对话器

- **底部**：输入框 + 快捷提示

## 场景 7：线索智能初筛 & 意向度评级

### 业务痛点
- 线索量大（百度推广、抖音、展会、转介绍），销售一个个聊太累
- 随手放弃很多潜力客户（"看着不像大客户"）
- 人工筛线索效率低、主观偏差大
- 高意向线索被淹没在低质量线索里，响应不及时

### AI 落地能力
- **自动判定**：基于客户咨询内容、行业属性、提问关键词
- **多维度评级**：
  - 意向度：高 / 中 / 低
  - 客户类型：快餐 / 火锅 / 食堂 / 酒店 / 创业
  - 预算层级：<3 万 / 3-5 万 / 5-10 万 / >10 万
  - 紧急度：急（下周开业）/ 中（1-3 个月）/ 缓（半年以上）
- **优先推送**：高意向线索自动推送给资深销售
- **AI 判定理由**：每条线索附带"为什么评这个等级"的解释

### Demo 展示形态
**界面**：单页工具——上传区 + 结果看板
- **上传区**：拖拽 Excel 或粘贴文本
- **结果看板**：
  - 统计卡片（总线索数 / S 级 / A 级 / B 级 / C 级）
  - 表格列表（每条线索的评级 + 标签 + AI 理由）
  - 筛选功能（按等级 / 行业 / 预算筛选）
- **导出**：一键导出分级后的 Excel

- 不同餐饮业态需要不同的设备配置方案

- 方案不专业、格式不统一，影响公司形象

  - 推荐方案（设备清单 + 配置说明）
  - 案例佐证（3-5 个同类客户案例）
  - 报价明细（总价、付款方式）

### 导航入口设计
- 顶部 Header：项目名称 + 返回首页
- 八大场景卡片网格（2 列桌面 / 1 列移动端）
- 每个卡片：场景编号 + 名称 + 一句话描述 + 标签（销售赋能/客户运营/资产沉淀）+ 进入按钮
- 底部：项目简介 + 联系方式

### Mock 数据策略
- 所有场景共享一套 Mock 数据（产品库、案例库、客户库）
- 数据存在 JSON 文件中，前端直接读取
- 演示时可以随时替换为真实 API
- 数据量控制在 50-100 条，保证加载速度

### 演示流程建议
1. **开场 2 分钟**：在导航页介绍项目定位和 8 大场景概览
2. **核心演示 10 分钟**：按优先级展示 2-3 个场景（建议：场景 6 → 场景 1 → 场景 2）
3. **收尾 2 分钟**：总结价值 + 下一步计划

## 落地优先级建议

### Phase 1：快速出 Demo（1-2 周）
**目标**：让老板/客户看到效果，争取预算
- 场景 6：AI 模拟培训（1.5 天）
- 场景 1：智能知识库（1.5 天）
- 场景 7：线索初筛（1 天）
- 导航入口 + 统一设计（1 天）
- **合计：5 天**

### Phase 2：完善核心场景（2-3 周）
**目标**：形成可对外展示的完整解决方案
- 场景 2：企微话术参谋（2-3 天）
- 场景 8：方案生成（2-3 天）
- 场景 4：复盘沉淀（1.5 天）
- **合计：6-7 天**

### Phase 3：系统对接（4-6 周）
**目标**：接入真实数据，内部试用
- 场景 3：SOP 跟进（需对接 CRM）
- 场景 5：数据应答（需对接 ERP/OMS）
- **合计：视系统对接复杂度而定**
