# CRM 转化数据驱动的场景优化方案

> 基于 `crm-conversion-analysis-2026-05-23.md` 中的真实 CRM 数据，
> 对 9 个 AI Sale 场景的阶段定义、话术策略、SOP 机制进行校准优化。
>
> 文档性质：优化提案，待落地执行。
> 来源数据：CRM/OA 线索转化截图手工整理（P1-P7 阶段漏斗 + 联系人/企业归因）。

## 二、场景 2 · 企微实时话术参谋（重点）

### 2.1 问题：阶段体系与现实脱节

**现状**：代码硬编码 `['认知','兴趣','评估','决策']` 4 阶段。

**数据冲突**：CRM 实际运营使用 **P1-P7** 七阶段，且 P1→P2、P2→P3 是最大流失段。

**执行项**：

- [ ] 将场景 2 的阶段体系从 4 段扩展为与 CRM 对齐的阶段模型（至少 5 段：P1 了解信息→P2 收集需求→P3 确认合作→P4 合同签署→P5-P7 后段）
- [ ] 改为可配置的阶段体系，支持不同客户的阶段映射差异
- [ ] 阶段命名统一到 CRM 术语，避免销售概念混淆

### 2.2 问题：缺少"滞留时间"驱动的风险提示

**现状**：风险信号只检测当场沉默≥5 分钟。

**数据**：P1 滞留率 59.83%（均值~12-14 天），P2 滞留率 45.48%（均值~8-10 天）。

**执行项**：

- [ ] 增加"超 7 天未推进"黄色提醒
- [ ] 增加"超 14 天未推进"红色警告 + 升级话术推荐
- [ ] 侧边栏展示"本客户在 P{n} 已停留 X 天，高于/低于同类客户均值"
- [ ] 超过 14 天的 P1/P2 机会自动触发"唤醒 SOP"或"退回线索池"建议

### 2.3 问题：话术推荐权重未使用真实转化率

**现状**：推荐度是静态预设值（95%/80%/70%），基于"历史 287 通成单数据"。

**数据**：各阶段转化率差异极大（P1→P2 仅 36.63%，P4→P5 高达 97.75%）。

**执行项**：

- [ ] 话术推荐权重引入阶段维度的转化率因子
- [ ] P1 阶段：需求挖掘类话术自动提权，资料投递类话术自动降权
- [ ] P2→P3 阶段：方案引导+试菜邀约类话术提权
- [ ] P4 之后：合同推进+催款类话术提权
- [ ] 推荐度改为动态计算，公式预留可调整的参数接口

### 2.4 问题：场景 2 只覆盖了售前

**现状**：认知→兴趣→评估→决策，只对应 P1-P4 前半段。

**数据**：P5→P6 转化率 72.61%，P5 滞留率 25.86%，说明后段仍有优化空间。

**执行项**：

- [ ] 评估是否将场景 2 延伸至 P5-P7（发货/回款/交付阶段）
- [ ] 如延伸，P5 增加发货确认/安装排期话术，P6 增加催款话术，P7 增加转介绍话术
- [ ] 如不延伸，在场景 2 侧边栏底部标注"当前客户已进入 P5+ 后段，建议切换到 OA 跟进视图"

### 2.5 问题：阶段可视化单薄

**现状**：只显示 1-2-3-4 圆点进度条。

**执行项**：

- [ ] 进度条改为带阶段名称+停留时间+转化率锚点的完整展示
- [ ] 每个阶段下方显示"同类客户平均停留时间"参考线
- [ ] 滞留超阈值的阶段高亮为黄色/红色

## 三、场景 1 · 销售随身知识库

### 3.1 问题：知识库未考虑线索来源差异

**数据**：市场投放占联系人 78.7%，但企业转化仅 4.4%；销售自拓占联系人 17.4%，企业转化达 12.6%。

**执行项**：

- [ ] 知识库应答策略分来源：投放线索→侧重公司背书+信任建设；自拓线索→可直接进入场景匹配
- [ ] FAQ 增加"来源归因"相关问答路径（如"我是百度看到的"→自动切换话术策略）

## 四、场景 3 · 客户分层 + SOP 跟进

### 4.1 问题：SOP 时间节点缺乏数据基准

**数据**：P1 滞留均值约 12-14 天，P2 滞留均值约 8-10 天。

**执行项**：

- [ ] Day 7（P1/P2 均值附近）→ 触发主动推进动作（方案推送/试菜邀约）
- [ ] Day 14（P1 滞留拐点）→ 触发升级机制（经理介入/退回线索池/沉睡标记）
- [ ] Day 30（超期）→ 自动标记沉睡，触发唤醒话术

### 4.2 问题：客户分层缺少"企业关联"维度

**数据**：企业层 79.5% 为"空"——联系人与企业关联缺失。

**执行项**：

- [ ] 分层标签增加"企业关联状态"维度（已关联/未关联/待确认）
- [ ] 未关联企业的联系人，AI 主动提示销售补充
- [ ] 联系人不关联企业则不能进入"S 级/高意向"分层

## 七、场景 6 · AI 新人模拟培训

### 7.1 问题：训练难度未对齐真实漏斗

**数据**：P1→P2 转化率 36.63%（最难），P4→P5 转化率 97.75%（最简单）。

**执行项**：

- [ ] 新增阶段维度的训练难度：P1→P2 设为"困难"模式（重点训练需求挖掘），P3→P4 设为"普通"模式
- [ ] 评分权重按阶段调整：在 P1 阶段，"需求挖掘力"权重 > "产品知识"权重
- [ ] 训练报告增加"阶段转化得分"维度，而不是只给总分

## 八、场景 7 · 线索初筛评级

### 8.1 问题：评级模型未纳入来源转化率

**数据**：市场投放企业转化率 4.4%，销售自拓企业转化率 12.6%。

**执行项**：

- [ ] 来源作为评级降权/加权因子：市场投放线索在企业关联前评级自动下调一档
- [ ] 销售自拓线索评级自动上调
- [ ] 评级系统增加"来源-意向偏差提醒"：当投放线索被评 S 级但未关联企业时，AI 标记为"待验证"

## 十、跨场景通用建议

### 10.1 阶段体系标准化

所有场景共用一套阶段定义，建议以 CRM 实际运营的 P1-P7 为基准：

```
P1 了解信息 → P2 收集需求 → P3 确认合作 → P4 合同签署 → P5 发货 → P6 已回款 → P7 已交付
```

各场景可从公共阶段定义中取子集（如场景 2 取 P1-P4，场景 7 取 P1-P2），但命名和边界一致。

### 10.2 滞留率纳入关键指标

将"各阶段>14 天滞留率"作为所有销售辅助功能的监控指标，在场景 2 侧边栏、场景 3 看板、场景 9 工作台统一展示。

### 10.3 数据归因缺口作为产品功能触发点

CRM 数据中联系人/企业关联断裂（79.5% 空）、来源分项之和≠总量等数据治理问题，应作为场景 5 和场景 3 的功能触发点——AI 在适当时机提示销售补充字段，而不是等后台管理员处理。
