## 2. 技术栈决策

| 场景建议 | 实际采用 | 说明 |
|---------|---------|------|
| React + Redux | Vanilla JS 模块化 | 无构建工具链，纯静态Demo |
| Node.js + PostgreSQL | 读JSON文件 | Demo阶段，不引入后端 |
| LLM AI层 | 规则引擎模拟 | AI文案用模板+变量填充 |
| Cron定时任务 | 按钮触发+相对时间计算 | Demo交互式触发 |

## 3. 设计偏差说明

**偏差1**: 标签体系从"5类规模"改为"数据驱动的动态标签"
- 原因：现有 `customers.json` 的 size 字段是字符串（如 "280平"），直接按数字范围分类不精确。改为规则引擎自动打标，规则可配置。

**偏差2**: 移除了"S/A/B/C/沉睡"分层标签
- 原因：P1-P7 阶段分类 + 健康度评分(0-100) 更精确。S/A/B/C 作为额外标签计算。
