# PRD：自动化 SOP 触达系统

> **版本**: v2.0（方向修正版） | **日期**: 2026-05-23
> **核心变更**: 从"销售看板手动操作"转为"系统自动触达、销售仅在异常节点介入"

---

## 0. 核心命题

**一句话定义**：一个**不需要销售手动操作**的客户跟进系统。系统根据客户所处的**生命周期阶段 + 时间周期**自动执行消息触达、阶段迁移、异常上报，销售只在系统解决不了的关键节点介入。

### 与 v1（CRM 看板方向）的根本区别

| 维度 | v1（错误方向） | v2（正确方向） |
|------|--------------|--------------|
| **消息谁发** | 销售看提示→复制文案→手动发 | 系统自动发送（企微/短信） |
| **阶段谁判断** | 销售看卡片→自己判断 | 系统根据规则自动判定+迁移 |
| **销售的任务** | 打开看板→选客户→操作 | 处理系统解决不了的异常 |
| **每日待办** | "今天该跟哪 5 个客户" | "今天有 2 件事需要你处理" |
| **系统输出** | 一个看板 | 一系列自动执行的动作 |

## 1. 用户故事

### 故事 1：一个客户从进入系统到成交的全自动流程

```
一个川菜馆老板（王老板）通过百度广告留下线索
  ↓
系统自动判定：川菜馆，280平，预算8-12万
  ↓  匹配SOP模板：标准餐饮客户SOP
  ↓
Day 1  10:00  系统自动发送欢迎消息（企微）
                "王老板您好！感谢您了解优特智厨……"
Day 2  10:00  系统自动推送方案（带客户案例）
                "根据您的场景，推荐YC-3000智能炒菜机……"
Day 4  10:00  客户未回复 → 系统自动发送第二次触达
                "王老板，这是附近川菜馆的实际使用数据……"
Day 7  10:00  客户回复"看着不错，怎么试菜？"
                ↓
               系统自动识别：关键词"试菜" → 阶段迁移 P1→P2
                ↓
               ⚡ 自动通知销售："王老板主动询问试菜，请安排"
                ⚡ 销售确认试菜时间后，系统自动发送确认消息
                ⚡ 试菜完成 → 销售标记"试菜完成" → 系统继续推进
                ↓
Day 14 10:00  系统自动发送促单消息
                "本月签约送3年延保，名额有限……"
Day 30        客户未推进 → 系统自动标记沉睡 → 发送唤醒消息
```

**关键观察**：销售只在"客户主动问试菜"这个节点介入一次，其余全部由系统自动完成。

### 故事 2：异常上报

```
客户回复"你们太贵了，XX品牌便宜2万"
  ↓
系统自动识别：关键词"太贵"+"竞品" → 异常级别 HIGH
  ↓
⚡ 即时上报给销售 + 推送应对话术
⚡ 销售介入处理，解决后标记"已处理"
⚡ 系统恢复自动推进
```

### 故事 3：多客户批量触达

```
销售团队有 200 个线索客户
  ↓
系统按各自的 SOP 时间线独立推进
  ↓
Day 1:  系统自动给所有"今日欢迎"的客户发送消息（约 15 条）
Day 3:  系统自动给所有"今日方案推送"的客户发送消息（约 12 条）
Day 7:  系统自动给所有"今日试菜邀约"的客户发送消息（约 8 条）
  ↓
销售打开系统，只看到 2 条需要他处理的异常：
  - "王老板主动问试菜，请确认时间"
  - "李总说太贵了，建议立即联系"
```

**关键观察**：200 个客户的日常跟进，销售只需要处理 2 个异常。这就是"系统承担 90% 工作"的含义。

## 2. 核心概念

### 2.1 客户生命周期阶段（P1-P7 + 规则驱动）

```
P1（了解信息）→ P2（收集需求）→ P3（确认合作）→ P4（合同签署）
→ P5（发货）→ P6（已回款）→ P7（已交付）

     ↕                          ↕
  P0（沉睡）               LOST（流失）
```

**阶段迁移方式**：

| 迁移 | 触发条件 | 自动/手动 |
|------|---------|----------|
| P1→P2 | 客户回复了"需求关键词"（面积/单量/人数等） | 自动 |
| P1→P0 | 超过 30 天无互动 | 自动 |
| P2→P3 | 客户主动问试菜/销售标记试菜完成 | 手动（销售介入） |
| P3→P4 | 客户确认合作意向 | 手动 |
| P4→P5 | 首付款到账（回调） | 自动 |
| P5→P6 | 尾款到账（回调） | 自动 |
| P6→P7 | 交付确认 | 手动 |
| 任意→LOST | 销售标记流失 | 手动 |

### 2.2 时间线规则（SOP 的原子单位）

每条规则定义：**在什么时间、什么条件下、做什么动作**

```json
{
  "ruleId": "R001",
  "name": "Day1 欢迎触达",
  "trigger": {
    "type": "stage_entry",     // 进入某阶段时触发
    "stage": "P1",
    "delay": "0d"              // 立即执行
  },
  "condition": null,           // 无条件
  "action": {
    "type": "send_message",
    "channel": "wecom",        // 企微/短信/邮件/系统通知
    "contentTemplate": "您好{customerName}，感谢您了解优特智厨！……"
  },
  "escalation": null           // 无异常上报
}
```

```json
{
  "ruleId": "R002",
  "name": "Day3 方案推送",
  "trigger": {
    "type": "stage_entry",
    "stage": "P1",
    "delay": "2d"
  },
  "condition": "stage == 'P1' && lastMessageStatus != 'replied'",  // 仅当客户未回复
  "action": {
    "type": "send_message",
    "channel": "wecom",
    "contentTemplate": "{customerName}，根据您的{customerType}场景，这是方案……"
  },
  "escalation": null
}
```

```json
{
  "ruleId": "R010",
  "name": "客户主动问试菜",
  "trigger": {
    "type": "keyword_match",   // 关键词匹配触发
    "keywords": ["试菜", "尝尝", "体验", "来看看"]
  },
  "condition": null,
  "action": {
    "type": "stage_transition",  // 阶段迁移
    "toStage": "P2"
  },
  "escalation": {
    "level": "MEDIUM",
    "message": "{customerName}主动询问试菜，请安排试菜时间",
    "assignTo": "sales"
  }
}
```

### 2.3 消息通道抽象

```
┌───────────────────────────────────────────┐
│           统一消息调度层                    │
│                                           │
│  ┌────────┐  ┌────────┐  ┌────────┐      │
│  │ 企微    │  │ 短信    │  │ 系统通知│      │

```

## 3. 功能需求

### 3.1 自动化消息触达引擎（核心）

- [ ] **时间线执行器**：每天定时扫描所有活跃客户的 SOP 时间线，执行到期任务
- [ ] **消息发送器**：调用消息通道发送消息，记录发送状态（成功/失败/已读/回复）
- [ ] **频率控制**：同一客户 24 小时内最多 2 条自动消息，避免骚扰
- [ ] **去重保护**：同一规则同一天不重复执行
- [ ] **失败重试**：发送失败自动重试 3 次（间隔 30 分钟）

### 3.2 阶段迁移引擎

- [ ] **关键词自动迁移**：客户回复特定关键词时自动推进阶段
- [ ] **时间阈值迁移**：超过指定天数无进展时自动降级/沉睡
- [ ] **销售手动迁移**：销售手动拖拽/点击迁移阶段
- [ ] **迁移广播**：阶段变更后触发对应的事件钩子（重新计算 SOP 时间线）

### 3.3 异常上报系统

- [ ] **关键词异常检测**：客户出现负面/竞品/价格敏感词时触发
- [ ] **超时异常检测**：SOP 步骤超过指定天数未完成时触发
- [ ] **上报分级**：LOW（系统自动处理）/ MEDIUM（通知销售）/ HIGH（即时电话通知）
- [ ] **处理闭环**：销售标记"已处理"后异常关闭，系统恢复自动推进

### 3.4 SOP 规则配置（管理后台）

- [ ] **规则 CRUD**：创建/编辑/启用/禁用 SOP 规则
- [ ] **触发条件配置**：阶段进入/时间延迟/关键词匹配/客户属性匹配
- [ ] **动作配置**：发送消息/迁移阶段/上报异常/更新标签
- [ ] **消息模板变量**：支持 {customerName} {customerType} {productName} 等变量
- [ ] **A/B 测试**：同一步骤可配置多版本文案，系统随机分配

### 3.5 销售工作台（销售看到的界面）

**不是看板，而是异常处理中心**：

```
┌─────────────────────────────────────────────┐
│  待处理 (2)               ← 只有异常才出现  │
│                                             │
│  🔴 王老板·主动问试菜                       │
│     系统建议：今天14:00前回电确认试菜时间    │
│     [确认时间] [转给同事] [稍后处理]         │
│                                             │
│  🟡 李总·反馈"太贵"                         │
│     系统建议：用人力成本换算回应             │
│     [查看话术] [标记已处理]                  │
│                                             │
├─────────────────────────────────────────────┤
│  今日自动触达 (15)          ← 灰度展示      │
│  系统今日自动发送了 15 条消息，无需你介入     │
│  [查看详情]                                 │
└─────────────────────────────────────────────┘
```

### 3.6 客户洞察（销售主动查看时）

- [ ] 客户当前阶段 + SOP 进度（已完成/待执行步骤）
- [ ] 互动时间线（系统自动发送记录 + 客户回复记录）
- [ ] 自动标签（行业/规模/意向/预算）
- [ ] 系统推荐的下一个最佳动作

## 4. 数据模型

### 4.1 规则模型

```typescript
interface SOPRule {
  ruleId: string;
  name: string;
  enabled: boolean;
  
  // 触发条件
  trigger: {
    type: 'stage_entry' | 'stage_exit' | 'keyword_match' | 'time_elapsed' | 'customer_attribute';
    stage?: string;
    delay?: string;           // "0d" / "2d" / "7d" / "30d"
    keywords?: string[];
    attributeCondition?: string;
  };
  
  // 前置条件（可选）
  condition?: string;         // 表达式："stage == 'P1' && lastMessageStatus != 'replied'"
  
  // 执行动作
  action: {
    type: 'send_message' | 'stage_transition' | 'tag_update' | 'escalation' | 'webhook';
    channel?: 'wecom' | 'sms' | 'system';
    contentTemplate?: string;
    toStage?: string;
    tags?: string[];
    escalationLevel?: 'LOW' | 'MEDIUM' | 'HIGH';
    escalationMessage?: string;
  };
  
  // 异常上报
  escalation?: {
    level: 'LOW' | 'MEDIUM' | 'HIGH';
    message: string;
    assignTo: 'sales' | 'manager';
    timeoutHours?: number;    // 销售超时未处理再次升级
  };
}
```

### 4.2 客户状态模型

```typescript
interface AutoCustomer {
  customerId: string;
  name: string;
  company: string;
  type: string;
  stage: string;            // P1-P7 / P0 / LOST
  stageEnteredAt: string;   // 进入当前阶段的时间
  
  // 自动标签
  autoTags: string[];
  
  // 消息记录
  messages: AutoMessage[];
  
  // 待处理异常
  activeEscalations: Escalation[];
  
  // SOP 时间线实例
  sopTimeline: SOPRuleInstance[];
}

interface AutoMessage {
  messageId: string;
  ruleId: string;
  channel: string;
  content: string;
  sentAt: string;
  status: 'sent' | 'delivered' | 'read' | 'replied' | 'failed';
  replyContent?: string;
  replyAt?: string;
}
```

### 4.3 SOP 时间线实例

```typescript
interface SOPRuleInstance {
  ruleId: string;
  name: string;
  scheduledAt: string;       // 计划执行时间
  executedAt?: string;       // 实际执行时间
  status: 'pending' | 'executed' | 'skipped' | 'failed';
  conditionMet: boolean;
  result?: string;
}
```

## 5. 验收标准

| 功能 | 验收项 | 优先级 |
|------|--------|-------|
| 时间线执行器 | 每天定时扫描所有活跃客户，在正确的时间执行正确的规则 | P0 |
| 消息发送 | 支持企微通道发送，带发送状态记录 | P0 |
| 频率控制 | 24h 内同一客户最多 2 条自动消息 | P0 |
| 去重保护 | 同一条规则同一天不重复执行 | P0 |
| 关键词迁移 | 客户回复"试菜"→ 自动 P1→P2 + 通知销售 | P0 |
| 关键词异常 | 客户说"太贵"→ 上报 + 推送应对话术 | P0 |
| 超时沉睡 | 超过 30 天无互动 → 自动标记 P0 + 发送唤醒 | P0 |
| 销售工作台 | 只展示需要销售处理的异常，不展示正常流程 | P0 |
| 规则配置 | 支持创建/编辑/启用/禁用 SOP 规则 | P1 |
| 消息模板变量 | 支持客户名称/类型/产品等变量替换 | P1 |

## 6. 范围边界

### 本版本包含
- 自动化消息触达引擎（核心）
- 阶段迁移引擎（关键词+时间阈值）
- 异常上报系统
- 销售工作台（异常处理视图）
- 客户互动时间线
- 从 `data/customers.json` 加载客户数据

### 本版本不包含
- 真实消息通道接入（Demo 用控制台日志 + 模拟发送）
- 规则配置界面的 UI（使用配置化 JS 模拟）
- 多销售角色/权限
- 数据持久化写入（只读 JSON）
