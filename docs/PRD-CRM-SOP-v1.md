## 0. 产品定位

### 0.1 一句话定义
面向商用厨具销售场景的**客户全生命周期管理 + AI 智能 SOP 跟进系统**，让销售知道「对谁、在什么时候、做什么动作」，系统自动提醒、自动推进、自动预警。

### 0.2 解决的核心问题

| 问题 | 现状 | 目标 |
|------|------|------|
| 跟进随性 | 销售凭感觉跟客户，挑着跟 | 系统编排 SOP，100% 覆盖 |
| 阶段不清晰 | 客户在哪个阶段凭记忆 | 可视化 P1-P7 全链路漏斗 |
| 遗忘流失 | 长决策周期客户易断联 | SOP 定时提醒 + 沉睡唤醒 |
| 管理层脱节 | 主管不知道下属客户状态 | 团队看板 + 异常预警一眼可见 |
| 数据孤岛 | 客户数据在销售脑子里带走 | 客户 360° 档案 + 互动时间线 |

### 0.3 目标用户角色

| 角色 | 核心诉求 | 使用频率 |
|------|---------|---------|
| **销售** | 知道今天该跟进谁、该说什么 | 每天多次 |
| **销售主管** | 掌握团队客户状态、SOP 执行情况 | 每天 1-2 次 |
| **运营/管理员** | 配置 SOP 模板、标签、阶段规则 | 定期维护 |

## 2. 功能需求详细说明

### 2.1 多视图客户管理

#### 2.1.1 卡片视图（默认首页）

**需求描述**：
客户以信息卡片形式排列，每张卡片展示核心信息，支持筛选和排序。

**卡片内容**：
- 客户名称 + 公司名
- 行业标签（川菜馆/快餐/食堂/酒店）
- 规模标签（面积）
- 当前阶段（P1-P7 带颜色+标签）
- 预算区间
- 负责销售
- 最近联系时间（相对时间："3天前"）
- 下次跟进动作 + 时间
- 健康度评分（0-100，带颜色指示）
- 标签列表
- 企业关联状态（是否关联企业）
- SOP 进度摘要（已完成/待完成步骤数）
- 快捷操作按钮（聊天/档案/方案）

**交互**：
- 点击卡片 → 展开客户 360° 详情（右侧抽屉或弹窗）
- 悬停 → 显示更多操作选项
- 筛选条件：阶段/行业/规模/意向/预算/销售负责人/状态

**验收标准**：
- [ ] AI 标签自动显示
- [ ] 阶段颜色正确
- [ ] 相对时间正确计算
- [ ] 点击可展开详情

#### 2.1.2 漏斗视图

**需求描述**：
以漏斗图表展示 P1→P7 阶段的客户分布数量和转化率。

**显示内容**：
- 每个阶段的客户数量（数字 + 横条宽度）
- 相邻阶段的转化率（百分比 + 箭头）
- 阶段滞留天数均值
- 点击某阶段可筛选该阶段的客户列表

**验收标准**：
- [ ] 漏斗数据与客户列表一致
- [ ] 转化率计算正确
- [ ] 点击筛选联动

#### 2.1.3 看板视图

**需求描述**：
看板式布局，每个阶段为一列，客户卡片可拖拽改变阶段。

**交互**：
- 拖拽卡片到下一阶段 → 弹出确认窗（选择阶段变更原因）
- 拖拽到"丢失"列 → 弹出流失原因表单

**验收标准**：
- [ ] 拖拽交互流畅
- [ ] 阶段变更记录时间线
- [ ] 丢失原因必填

#### 2.1.4 客户 360° 详情

**需求描述**：点击客户卡片展开的完整档案视图。

**内容区域**：
1. **基本信息区**：名称、公司、联系信息、来源渠道
2. **标签区**：显示所有标签，可手动添加/移除
3. **阶段信息区**：当前阶段、阶段停留天数、阶段预警状态
4. **订单信息区**：关联订单、设备配置、总金额
5. **SOP 时间线区**：全部 SOP 步骤的完成状态（已完成/待执行/已逾期）
6. **互动历史区**：全部互动记录的时间轴
7. **同类型客户参考**：展示 2-3 个同类已成交客户的参考信息

**验收标准**：
- [ ] 所有区域数据正确
- [ ] SOP 时间线可视化
- [ ] 互动历史按时间排序

### 2.2 SOP 智能引擎

#### 2.2.1 SOP 模板

**需求描述**：SOP 跟进模板定义为客户跟进的标准流程。

**模板结构**：
```json
{
  "templateId": "SOP-CANTEEN-001",
  "name": "食堂客户跟进SOP",
  "description": "面向企业食堂/团餐客户的标准化跟进流程",
  "applicableTo": {
    "types": ["CANTEEN","CATERING"],
    "stages": ["P1","P2","P3"],
    "minBudget": 100000
  },
  "steps": [
    {
      "id": 1,
      "name": "欢迎触达",
      "delay": {"type": "immediate"},
      "actionType": "send_message",
      "content": "您好，感谢了解优特智厨！...",
      "channel": "wecom"
    }
  ]
}
```

**验收标准**：
- [ ] 支持条件匹配（按客户类型/阶段/预算匹配）
- [ ] 延迟时间支持 immediate / hours:N / days:N
- [ ] 每个步骤支持前置条件
- [ ] 支持超时升级机制

#### 2.2.2 阶段转移机（Pipeline Engine）

**需求描述**：客户在 P1-P7 阶段间的自动/手动流转。

**阶段定义**：
| 阶段 | 标签 | 颜色 | 说明 |
|------|------|------|------|
| P1 | 了解信息 | 灰色 | 首次接触，获取资料 |
| P2 | 收集需求 | 蓝色 | 已获取场景信息 |
| P3 | 确认合作 | 黄色 | 试菜完成，意向确认 |
| P4 | 合同签署 | 橙色 | 进入合同流程 |
| P5 | 发货 | 紫色 | 合同执行 |
| P6 | 已回款 | 绿色 | 全部款项到账 |
| P7 | 已交付 | 绿色深色 | 安装交付完成 |
| LOST | 已丢失 | 红色 | 客户流失 |

**转移条件**：
- P1→P2：手动推进（销售确认已收集需求）
- P2→P3：客户确认试菜时间自动触发
- P3→P4：手动推进（客户确认合作意向）
- P4→P5：首付款到账自动触发
- P5→P6：尾款到账自动触发
- P6→P7：交付确认签字

**滞留预警阈值**：
- P1 滞留 >7天 → 黄色预警
- P1 滞留 >14天 → 红色预警 + 升级主管
- P2 滞留 >10天 → 黄色预警
- P3 滞留 >14天 → 黄色预警
- 任意阶段 >30天无互动 → 自动标记沉睡

**验收标准**：
- [ ] 阶段转移记录时间线
- [ ] 滞留预警正确触发
- [ ] 橙色预警升级主管

#### 2.2.3 自动打标规则引擎

**需求描述**：基于客户属性和行为自动计算标签。

**内置规则**：
| 规则 | 条件 | 自动标签 |
|------|------|---------|
| 高意向 | 主动问试菜 + 预算明确 + 有开业时间 | "高意向" |
| 比价中 | 提到竞品名称 | "比价中" |
| 沉睡预警 | 超过14天未联系，P1-P3阶段 | "沉睡预警" |
| 大客户 | 预算 >30万 或 连锁 >5家门店 | "大客户" |
| 试菜完成 | 互动记录中有试菜 action | "已试菜" |

**验收标准**：
- [ ] 客户加载时自动计算标签
- [ ] 标签支持叠加
- [ ] 预警标签带标记指示

### 2.4 AI 智能能力

#### 2.4.1 文案生成

**功能描述**：基于客户信息和当前 SOP 步骤，AI 生成 3 个版本的跟进文案。

- 版本 A：标准版（稳妥）
- 版本 B：案例驱动版（引用同类型客户）
- 版本 C：紧迫感版（限时活动）

#### 2.4.2 客户健康度评分

**评分维度**：
- 最近互动时间（30分）：越近越高
- 阶段进展（25分）：越往后越高
- 互动数量（20分）：适度频率最高
- 情绪倾向（15分）：正向倾向高
- 预算明确度（10分）：越明确越高

#### 2.4.3 最佳下一步推荐

基于客户当前状态，AI 推荐最有效的下一步动作。

## 3. 数据模型

### 3.1 客户数据模型

```typescript
interface Customer {
  customerId: string;          // 唯一ID: CUS-2026-0001
  name: string;                // 客户名称
  company: string | null;      // 公司名
  contact: string;             // 联系人
  type: string;                // 行业类型中文
  typeCode: string;            // 行业类型编码
  region: string;              // 所属大区
  city: string;                // 城市
  storeCount: number;          // 门店数
  budget: string;              // 预算区间
  stage: string;               // P1-P7 / LOST
  stageLabel: string;          // 阶段中文名
  lastContact: string;         // 最后联系日
  nextAction: string | null;   // 下次跟进日
  nextActionLabel: string;     // 下次动作描述
  status: string;              // active / sleeping / closed
  source: string;              // 来源渠道
  tags: string[];              // 标签列表
  healthScore: number;         // 健康度 0-100
  history: Interaction[];      // 互动历史
  order: Order | null;         // 关联订单
  decisionMaker: string;       // 决策角色编码
  decisionMakerLabel: string;  // 决策角色
}

interface Interaction {
  date: string;
  action: string;
  detail: string;
}

interface Order {
  id: string;
  product: string;
  amount: number;
  status: string;
}
```

### 3.2 SOP 数据模型

```typescript
interface SOPTemplate {
  templateId: string;
  name: string;
  description: string;
  applicableTo: {
    types: string[];
    stages: string[];
    minBudget: number | null;
    maxBudget: number | null;
  };
  steps: SOPStep[];
}

interface SOPStep {
  id: number;
  name: string;
  delay: { type: 'immediate' | 'hours' | 'days'; value?: number };
  condition?: string;           // 前置条件
  actionType: string;           // send_message / send_proposal / invite_trial /等等
  content: string | string[];   // 文案内容
  channel?: string;             // wecom / phone / sms
  escalation?: {                // 超时升级
    timeout: { type: 'hours' | 'days'; value: number };
    action: string;
    message: string;
  };
  autoReminders?: { delay: { type: 'hours' | 'days'; value: number }; type: string }[];
}

interface SOPInstance {
  customerId: string;
  templateId: string;
  steps: SOPInstanceStep[];
  appliedAt: string;
  status: 'active' | 'paused' | 'completed';
}

interface SOPInstanceStep {
  stepId: number;
  status: 'pending' | 'done' | 'skipped' | 'overdue';
  dueAt: string | null;
  completedAt: string | null;
  completedBy: string | null;
}
```

## 4. 非功能性需求

### 4.1 性能
- 客户列表加载 < 500ms（100条以内）
- 筛选操作响应 < 200ms
- AI 文案生成 < 2s（模拟延迟 600-1200ms）

### 4.2 可用性
- 所有操作 3 步内完成
- 关键操作（完成/推进）带 Toast 反馈
- 空数据 / 加载 / 错误 三种状态全覆盖

### 4.3 兼容性
- 支持 Chrome / Edge / Firefox 最新版本
- 适配 1280px+ 桌面分辨率
- 移动端不做强要求（B端管理后台）

### 4.4 数据一致性
- 客户数据修改后，漏斗/筛选/看板同步更新
- SOP 步骤完成 → 自动推进到下一条
- 阶段变更 → 触发对应规则
