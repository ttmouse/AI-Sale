# 企业微信模拟界面设计原则

> 适用范围：scene-12-workbench 及其他企业微信 UI 模拟页面

## 1. 真实性优先

- 聊天气泡上**不显示"已读"** —— 官方企业微信无此功能
- 消息状态（已送达 / 已读）只在会话列表页展示，不在单聊气泡内

## 2. 深色视觉容器

手机区域用深色背景包裹，替代投影效果：

| 层级 | 色值 | 用途 |
|------|------|------|
| 手机本体 | `#000` | 模拟设备黑边框 |
| 容器背景 | `#2a2a2a` | 包裹手机的区域衬底 |
| 控件背景 | `#1e1e1e` | 播放控制栏等附属控件 |
| 深色边框 | `#3a3a3a` | 深色区域的元素分隔线 |
| 文字颜色 | `#ccc` / `#888` | 深色区域内的文字 |
| 滚动条 | `#444` | 深色区域滚动条 |

容器背景色至少 `#1a1a1a` 以上，与手机纯黑保持可辨识的层次。

## 3. 间距规则

- 手机顶部与深色容器留 `8px`，不与容器顶部紧贴
- 播放控制栏用 `0.5rem` padding，与手机边缘对齐

## 4. 主题适配

- 深色区域内的所有元素（控制栏、按钮、进度条、滚动条）颜色都要适配深色主题
- **不要**用浅色主题的 CSS 变量（`var(--bg-surface)`、`var(--border)`、`var(--text-muted)` 等）——这些在亮色区域是正确的，在深色区域是突兀的
- 浅色区域（左右侧面板、时间轴）继续使用主题 CSS 变量，不受影响

## 5. 版面空间

- 导航栏和阶段 Tab 合并为单行
- 手机尺寸：375×812（iPhone mini 标准）
- 主内容区用 `flex: 1` + `grid-template-rows: 1fr` 填充剩余视口，不产生底部空白

## 6. 滚动条规范

| 区域 | 宽度 | 颜色 |
|------|------|------|
| 浅色面板（左/右/时间轴） | 3px | `var(--border)` |
| 深色区域（中间） | 3px | `#444` |
