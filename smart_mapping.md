# Smart Mapping — 动态 Prompt 生成

> 不是静态提示词，是「根据项目规则 + 当前上下文 → 组装最合适的 prompt」的系统。
> 基于日志中重复的模式自然进化，结构由项目自己长出来。

---

## 项目 DNA

```
行业：商用厨具（智能炒菜机 / 蒸柜 / 油炸机）
产品线：YC-1000/2000/3000 系列 + ZNG/YZG/YKX 等 16 SKU

关键转化杠杆：试菜（成交客户 78% 经历过）、案例佐证、人力成本换算法
```

## 日志中发现的常见陷阱

> 从 `smart_mapping_log.md` 同步，正文只收录重复 2+ 次的模式

- **路径错位**：`tokens.css` 在 `shared/` 但多场景引用为 `shared/css/` — 涉及文件引用时要验证实际位置
- **AI 回复与数据脱节**：当前所有场景的 AI 输出是硬编码文本，不读 JSON — 需要时先加载对应 JSON，再用数据生成回复
- **跨场景无状态**：同一客户在不同场景间不共享上下文 — 生成涉及客户的数据时要说明"数据来源：XXX.json"

## AI 层架构决策

```
当前所有场景的 AI 层是「规则引擎」不是 LLM
├─ 输入：用户文本
├─ 处理：关键词匹配（lower.includes + if/else 链）
├─ 输出：预制格式化文本（带 emoji + 分隔线）
└─ 数据源：独立 fetch 加载 data/*.json → JS 格式化

如果未来接入真 LLM，需要：
  1. 把规则引擎作为 fallback（0 延迟、可预期）
  2. LLM 负责：模糊匹配、意图消歧、多轮对话
  3. 规则引擎负责：精确查询、格式化输出、价格计算
```

这个决策影响 prompt 生成器中的 `输出规范` 字段——当前所有场景写的是「LLM 输出规则」，实际运行时是 JS 规则引擎输出。如果需要真 LLM，prompt 生成器需要加一个 `ai_layer` 参数。

## 数据层结构笔记

| 文件 | 根结构 | 关键字段 | 注意点 |
|------|--------|---------|--------|
| products.json | `.products[]` | price 为 number（元），scenes 为中文数组，customerTypes 为英文枚举值 |
| customers.json | `.customers[]` | stage 为 P0-P5，tags 为中文，history 为数组 |
| cases.json | `.cases[]` | totalPrice 为 number（元），products 为字符串数组 |
| dict.json | `.{xxx}.values[]` | 双层结构：`dict.customerTypes.values`，code 为英文，label 为中文 |

## Prompt 生成器

### 场景上下文模板

```
你正在处理 AI-Sale Demo 系统的 {scene_name}。
这是一个 {scene_description}。

当前数据上下文：
- 客户：{customer_context}
- 产品：{product_context}  
- 阶段：{stage_context}

输出规范：
{output_rules}

约束：
- 不要胡编数据，优先使用 data/*.json 中已有的内容
- 如果数据不足以回答，输出降级路径（转人工 / 提交工单）
{extra_constraints}
```

### 输出规则仓库（按场景组装）

```
通用规则：
- 金额一律 ¥X.XX 万 或 ¥X,XXX，千分位
- 时间用相对时间（X 分钟前 / 今天 / 昨天 / 3 天前）
- 数据来源标注小字（📄 产品手册 / 📊 案例库 #ID / 🔗 ERP 订单系统）

S1 额外规则：
- 答案底部必须带引用来源
- 参数用结构化卡片（表格）展示
- 不确定时输出推荐转人工路径

S2 额外规则：
- 输出 3 个话术选项，每项带推荐度百分比
- 每项话术下方带「推荐理由」
- 顶部显示客户阶段判断 + 意图识别

S3 额外规则：
- 客户卡片含未读数、阶段停留天数、健康度评分
- 超 14 天未推进 → 红色预警

S4 额外规则：
- 分析报告含成功因素 / 流失点 / 改进建议
- 高转化话术可一键入库

S5 额外规则：
- 每条数据标注系统来源（ERP/OMS/CRM）
- 接入异常时显示缓存数据时间 + 重试按钮

S6 额外规则：
- 每题实时打分（0-10）+ 一句话点评
- 对练结束六维雷达图（开场/挖需/价值/异议/临门/节奏）

S7 额外规则：
- 每条线索带评级依据（意向/预算/类型/完整度加权）
- 统计看板含来源分布

S8 额外规则：
- 必须输出 3 个方案对比（经济/标准/高配）
- 方案中嵌入同类型客户案例引用
- 含 ROI 测算（回本周期）
```

## 使用方式

```
1. 遇到具体任务时，确定涉及的场景编号
2. 从「场景上下文映射表」取参数
3. 从「输出规则仓库」取该场景的规则
4. 组装成完整 prompt（场景上下文模板 + 规则）
5. 如需验证数据，读 data/*.json / docs/*.md
6. 完成任务后，如果有新发现 → 记到 smart_mapping_log.md
7. 如果某个发现重复出现 → 升格到这个文件的对应章节
```
